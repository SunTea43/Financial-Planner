# Commit Guidelines

This project uses a Conventional Commits style.

Format:

```text
<type>: <description>
```

Allowed types:

- feat
- fix
- docs
- style
- refactor
- perf
- test
- chore
- ci

Rules:

- Use imperative present tense.
- Start description in lowercase.
- Do not end title with a period.
- Keep title line at 72 chars or less.

Example:

```text
feat: add monthly net worth comparison report
```

For complete rules and examples, see COMMIT_GUIDELINES.md in the project root.
